## Upgrade chain

Each upgrade's ticket references the immediately-prior package id. Type
identity sticks with the version where a module was first published.

| Version | Package ID | Notes |
|---|---|---|
| v1 (2026-03-12) | `0x6d968a4810498a16f18beca840667350ed5aafc119c3d8cbd3033676b622c774` | Original publish. `market_factory`, `yes_token`, `no_token`. |
| v2 (2026-05-13) | `0xf2189af211ace44b5365d3c0cb8a8b96137897957fa53b7f83570db0f1f10068` | Added `agent_cap` module. **Type origin for agent_cap lives here.** |
| v3 (2026-05-14) | `0x50a58add3954967d6c6480469b9fa78f3f7bb21fed9cda88323cdf7a87771c29` | `agent_cap::create` switched to `transfer::share_object` so the agent can reference the cap. |

<Callout type="warning">
  Before each upgrade, bump `published-at` in `Move.toml` to the **latest**
  package id. Otherwise `sui client upgrade` errors with
  `PackageIDDoesNotMatch`.
</Callout>
