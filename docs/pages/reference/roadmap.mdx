# Roadmap

DeepMarket is built to ship and keep shipping. This page tracks what's
verified live, what's actively in progress, and what's parked for later.

## Shipped & verified live

These are running on Sui testnet today and have been exercised end-to-end:

- **Spot YES/NO markets** — create / mint / trade / resolve / redeem
- **DeepBook Predict full lifecycle** — manager init → deposit → mint →
  redeem → withdraw
- **Per-market encrypted chat** — Sui Stack Messaging + Walrus + Seal,
  with one-popup auto-sign and an invite chain
- **Telegram trader bot** — per-user custodial wallets, autonomous
  60-second strategy loop, auto-redeem on settlement
- **LLM agent** — Claude decisions (Opus 4.7), batched 1-3 mints per tick,
  win-rate-aware sizing circuit breaker, structured tool-use output
- **MemWal semantic memory** — every settled trade written as a Walrus
  blob, vector-recalled into the prompt at decision time
- **AgentCap** — on-chain Move policy object + audit log; create, discover,
  record_decision, and event-stream-rendered decision log all verified

## Active work

1. **AgentCap revoke flow end-to-end test** — code path exists, awaits a
   click-through to confirm on-chain. Quick.
2. **TEE attestation** — wrap the Claude decision call in a Trusted
   Execution Environment so the model + prompt are independently
   verifiable. Forking `nautical` as the base.
3. **Mainnet readiness pass** — Spot contracts + AgentCap are ready to
   redeploy; Predict integration follows Mysten's mainnet launch.

## Near-term polish

- **Tokenized share token on `PredictManager`** — wrap manager positions
  in a transferable share. Unlocks composability with margin, lending,
  vault aggregators, and structured products.
- **Range positions panel** — UI list, gated on the Predict server
  exposing `/managers/:id/range-positions/summary` (currently 404).
- **Multi-oracle agent context** — the agent currently sees one oracle
  per tick; show it all active oracles so it can pick the best setup.
- **Auto-deposit only the gap** — the bot currently deposits full
  `quantity` per mint, wasting a few cents in dUSDC churn.
- **Down-direction strategy mode** — the rule-based fallback always mints
  UP. The LLM agent already picks both directions.

## Larger initiatives

- **Vol-arb signal feed** — back-solve Predict's implied vol from
  `OracleSVI`, compare against external option venues (Deribit / Polymarket
  smile), surface mispricings as agent input. Start signal-only, then
  optionally execute the Predict leg.
- **Surface Studio** — live 3D BTC vol surface (strike × expiry → IV)
  streamed from `OracleSVIUpdated` events, with arbitrage-free checks for
  butterfly / calendar violations.
- **PLP risk dashboard** — vault utilization, withdrawal-limiter token-
  bucket state, per-oracle exposure, ±5σ what-if PnL simulator.
- **zkLogin in the chat relayer** — Slush / Google sign-in is currently
  rejected by the upstream relayer (`Unknown signature scheme flag: 0x05`).
- **Mobile-first PWA** — install-to-home, push notifications via the
  Telegram bot's existing alert pipeline.
